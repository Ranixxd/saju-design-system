# Toast

사용자 액션에 대한 간결한 피드백 메시지를 일시적으로 표시합니다.


## ✅ Do

- 사용자 액션의 **결과 피드백**에 사용하세요. (저장됨, 오류 발생 등)
- 메시지는 **짧고 명확하게** 작성하세요. (1~2문장)
- `duration`으로 자동 사라지는 시간을 상황에 맞게 설정하세요.
- 여러 Toast가 동시에 나타날 수 있으면 스택 형태로 관리하세요.
- 화면 모서리(우측 하단 권장)에 일관되게 배치하세요.

## ❌ Don't

- 중요한 결정이나 확인이 필요한 경우에는 Modal을 사용하세요.
- Toast에 긴 텍스트나 복잡한 액션을 담지 마세요.
- 연속으로 너무 많은 Toast를 띄우지 마세요. (3개 이상 동시 노출 지양)
- 오류 메시지를 Toast로만 전달하지 마세요. 폼 오류는 인라인으로도 표시해야 합니다.
- `duration`을 너무 짧게 (2초 이하) 설정하지 마세요.


## Variant 사용 기준

| Variant | 사용 예 |
|---------|--------|
| `success` | 저장 완료, 업로드 성공 |
| `danger` | 오류 발생, 삭제 실패 |
| `warning` | 주의 필요, 만료 예정 |
| `info` | 일반 안내, 업데이트 알림 |


## duration 가이드

| 상황 | 권장 duration |
|------|-------------|
| 짧은 성공 메시지 | 3000ms |
| 일반 안내 | 4000ms |
| 오류/경고 | 5000ms 이상 또는 수동 닫기 |


## 접근성

- `role="status"` (info/success) 또는 `role="alert"` (warning/danger)를 사용하세요.
- `aria-live="polite"` 또는 `aria-live="assertive"`로 스크린리더에 알리세요.
- 사용자가 직접 닫을 수 있는 닫기 버튼을 항상 제공하세요.
- 자동으로 사라지는 Toast는 호버 시 타이머가 멈춰야 합니다.
