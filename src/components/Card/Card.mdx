# Card

관련된 정보를 하나의 컨테이너로 묶어 표현하는 레이아웃 컴포넌트입니다.


## ✅ Do

- 연관된 정보를 하나의 단위로 묶을 때 사용하세요.
- `hoverable`은 클릭 가능한 카드에만 사용하세요. (사용자에게 인터랙션 기대를 심어줍니다)
- 카드 내부 레이아웃은 일관된 패딩 기준을 유지하세요.
- 클릭 가능한 카드 전체 영역에 `role="button"` 또는 `<a>`를 사용하세요.

## ❌ Don't

- 카드 안에 카드를 중첩하지 마세요. (레이아웃 복잡도 증가)
- 클릭 이벤트 없는 카드에 `hoverable`을 사용하지 마세요.
- `shadow`와 `bordered`를 같이 강하게 사용하면 너무 무거워 보입니다.
- 카드 내부에 너무 많은 정보를 담지 마세요. 스캔 가능한 수준을 유지하세요.


## shadow + bordered 조합 가이드

| 상황 | 권장 설정 |
|------|----------|
| 기본 콘텐츠 카드 | `shadow="sm"` + `bordered` |
| 강조 카드 | `shadow="md"` + `bordered={false}` |
| 배경과 구분 충분할 때 | `shadow="none"` + `bordered` |
| 플로팅 카드 | `shadow="lg"` + `bordered={false}` |


## 접근성

- 클릭 가능한 카드는 `role="article"` 또는 `<article>`로 감싸고 포커스 가능하게 만드세요.
- 카드 내 주요 정보(제목)를 가장 먼저 배치해 스크린리더 탐색을 돕습니다.
- `hoverable` 카드는 키보드 포커스 시에도 동일한 시각적 피드백을 제공하세요.
