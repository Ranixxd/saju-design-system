# Badge

상태, 카운트, 분류를 한눈에 전달하는 레이블입니다.

---

## Variants

| Variant | 의미 |
|---------|------|
| `neutral` | 일반 정보, 기본 상태 |
| `primary` | 신규, 강조할 정보 |
| `success` | 완료, 성공, 활성 |
| `warning` | 주의, 검토 필요 |
| `danger` | 오류, 실패, 긴급 |
| `info` | 안내, 진행 중 |

---

## ✅ Do

- 상태를 **짧고 명확하게** 표현하세요. 1~3단어가 적당합니다.
- 동일한 컨텍스트에서는 variant를 **일관되게** 사용하세요. (완료 = 항상 success)
- 숫자 카운트(알림 수, 미처리 건수)에도 활용할 수 있습니다.
- 상태 변화가 있는 항목 옆에 `dot` 배지로 시선을 유도하세요.

## ❌ Don't

- 긴 문장을 Badge에 넣지 마세요. 텍스트가 길면 가독성이 떨어집니다.
- 한 줄에 Badge를 **3개 이상** 나열하지 마세요.
- 색상만으로 의미를 구분하지 마세요. 색맹 사용자를 위해 텍스트도 함께 제공하세요.
- 클릭 가능한 요소처럼 보이게 스타일을 추가하지 마세요. 인터랙션이 필요하다면 Tag를 사용하세요.

---

## Badge vs Tag 선택 기준

| 상황 | 권장 컴포넌트 |
|------|-------------|
| 읽기 전용 상태 표시 | Badge |
| 사용자가 제거 가능 | Tag |
| 클릭으로 필터링 | Tag |
