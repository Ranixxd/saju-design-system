# Button

액션을 트리거하는 가장 기본적인 인터랙티브 요소입니다.

---

## Variants

| Variant | 용도 |
|---------|------|
| `primary` | 페이지의 핵심 액션 (저장, 제출, 다음) |
| `secondary` | 보조 액션, primary와 나란히 배치 |
| `outline` | 중간 강도의 액션 |
| `ghost` | 최소한의 강조가 필요한 액션 |
| `danger` | 삭제 등 되돌릴 수 없는 액션 |

---

## ✅ Do

- **Primary 버튼은 한 페이지(또는 한 섹션)당 하나만 사용하세요.** 두 개 이상이면 사용자가 무엇을 눌러야 할지 혼란스럽습니다.
- 버튼 텍스트는 **동사로 시작**하세요. (저장하기, 확인, 삭제, 다음으로)
- Primary + Secondary처럼 **위계가 명확한 쌍**으로 배치하세요.
- 아이콘은 텍스트와 함께 사용해 의미를 명확히 하세요.
- `loading` prop으로 비동기 액션 진행 상태를 사용자에게 알려주세요.

## ❌ Don't

- 같은 영역에 Primary 버튼을 **2개 이상** 두지 마세요.
- 버튼 텍스트를 **4단어 이상**으로 길게 쓰지 마세요.
- `danger` 버튼은 확인 모달 없이 단독으로 배치하지 마세요.
- `disabled` 상태만 보여주고 왜 비활성인지 사용자에게 알려주지 않는 건 피하세요.
- 링크 이동 목적이라면 Button 대신 `<a>` 태그를 사용하세요.

---

## 사이즈 선택 기준

| Size | 사용 상황 |
|------|----------|
| `sm` | 테이블 내부, 태그 옆, 인라인 액션 |
| `md` | 폼, 카드, 모달 내 기본값 |
| `lg` | 히어로 섹션, 랜딩 페이지 CTA |

---

## 접근성

- `loading` 상태에서 자동으로 `aria-busy="true"`가 설정됩니다.
- 아이콘만 있는 버튼은 반드시 `aria-label`을 제공하세요.
- 키보드 포커스 스타일이 기본 제공됩니다 (`:focus-visible`).
