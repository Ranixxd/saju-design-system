# Button

사용자의 액션을 유도하는 가장 기본적인 인터랙티브 요소입니다.


## Variant 사용 기준

| Variant | 용도 |
|---------|------|
| `primary` | 페이지에서 가장 중요한 액션 1개 (예: 저장, 제출) |
| `secondary` | 보조 액션 (예: 취소, 뒤로가기) |
| `outline` | primary와 나란히 쓸 때 시각적 위계 구분 |
| `ghost` | 툴바·카드 내부 등 배경과 어우러져야 할 때 |
| `danger` | 삭제·탈퇴 등 되돌릴 수 없는 파괴적 액션 |


## ✅ Do

- 한 화면에 `primary` 버튼은 **1개**만 사용하세요.
- 버튼 텍스트는 **동사형**으로 작성하세요. (예: "저장하기", "계속하기")
- 로딩 중에는 `loading` prop을 사용해 중복 제출을 방지하세요.
- 아이콘만 있는 버튼에는 반드시 `aria-label`을 제공하세요.

## ❌ Don't

- 한 줄에 `primary` 버튼을 2개 이상 배치하지 마세요.
- "클릭", "여기"처럼 맥락 없는 텍스트를 사용하지 마세요.
- `danger` 버튼은 확인 모달 없이 바로 실행되게 하지 마세요.
- `disabled` 상태의 버튼에 툴팁 없이 이유를 알 수 없게 하지 마세요.


## Size 선택 기준

| Size | 사용 맥락 |
|------|----------|
| `sm` | 테이블 행 내부, 컴팩트 UI |
| `md` | 기본값, 대부분의 상황 |
| `lg` | 랜딩 페이지 CTA, 모달 하단 |


## 접근성

- `loading` 상태에서 `aria-busy="true"`가 자동으로 설정됩니다.
- `disabled` 버튼은 포커스를 받지 않으므로, 이유 안내가 필요하면 래퍼에 툴팁을 추가하세요.
- `leftIcon` / `rightIcon`은 `aria-hidden="true"`로 처리하세요.
