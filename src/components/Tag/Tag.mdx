# Tag

사용자가 추가하거나 제거할 수 있는 레이블입니다.

---

## ✅ Do

- 사용자가 **직접 추가/삭제**할 수 있는 항목에 사용하세요. (관심사 태그, 검색 필터)
- `onRemove`를 제공해 사용자가 태그를 지울 수 있게 하세요.
- 입력 필드와 함께 조합해 태그 입력 UI를 만드세요.
- 태그 텍스트는 짧고 명확하게 유지하세요. (1~3단어)

## ❌ Don't

- 읽기 전용 상태 표시에는 Tag 대신 Badge를 사용하세요.
- 한 영역에 태그를 **10개 이상** 나열하면 시각적으로 복잡해집니다. 줄 바꿈 처리나 "더 보기"를 고려하세요.
- 삭제 버튼(`onRemove`) 없이 interactive한 것처럼 스타일을 주지 마세요.

---

## Badge vs Tag 선택 기준

| 상황 | 권장 컴포넌트 |
|------|-------------|
| 읽기 전용 상태 표시 | Badge |
| 사용자가 추가/제거 가능 | Tag |
| 클릭으로 필터 토글 | Tag |

---

## 접근성

- 삭제 버튼에 `aria-label="Remove tag"`가 자동 설정됩니다.
- 키보드로 삭제 버튼에 포커스 후 Enter/Space로 제거할 수 있습니다.
