# Modal

사용자의 주의가 필요한 작업이나 정보를 오버레이로 표시합니다.

---

## ✅ Do

- **사용자의 즉각적인 반응이 필요한 경우**에만 사용하세요. (삭제 확인, 중요 알림)
- `title`을 항상 제공해 사용자가 무엇을 해야 하는지 명확히 하세요.
- `footer`에 액션 버튼을 배치하고, 취소/확인 순서를 일관되게 유지하세요. (취소 → 확인)
- ESC 키와 오버레이 클릭으로 닫을 수 있도록 기본값을 유지하세요.
- 파괴적 액션(삭제)은 `danger` 버튼으로 명확히 표시하세요.

## ❌ Don't

- 단순한 정보 표시에는 Modal 대신 Toast나 Tooltip을 사용하세요.
- Modal 위에 Modal을 띄우지 마세요. (중첩 Modal)
- 한 번에 여러 개의 Modal을 열지 마세요.
- 자동으로 닫히는 Modal을 사용하지 마세요. 사용자가 내용을 읽기 전에 닫힐 수 있습니다.
- Modal 안에 너무 많은 폼 필드를 넣지 마세요. 별도 페이지로 이동하는 것이 낫습니다.

---

## 사이즈 선택 기준

| Size | 사용 상황 |
|------|----------|
| `sm` | 확인/경고 다이얼로그 (텍스트 1~2줄) |
| `md` | 기본값, 짧은 폼, 상세 정보 |
| `lg` | 복잡한 폼, 미리보기 |
| `xl` | 대시보드, 에디터 |
| `full` | 전체 화면이 필요한 복잡한 작업 |

---

## 접근성

- 열릴 때 포커스가 Modal 내부로 자동 이동해야 합니다.
- `aria-modal="true"`, `role="dialog"`, `aria-labelledby`가 자동 설정됩니다.
- Modal이 열린 동안 배경 스크롤은 자동으로 잠깁니다.
- ESC 키로 닫기가 기본 지원됩니다.
