# Modal

현재 작업을 중단하고 사용자의 즉각적인 주의나 결정이 필요할 때 사용하는 오버레이 컴포넌트입니다.


## ✅ Do

- **중요한 결정**이나 **작업 완료 확인**에 사용하세요.
- `title`로 모달의 목적을 명확히 하세요.
- `footer`에 행동 버튼(확인/취소)을 배치하세요.
- Esc 키(`closeOnEscape`)와 오버레이 클릭(`closeOnOverlayClick`)으로 닫을 수 있게 하세요.
- 모달이 열리면 포커스를 모달 내부로 이동시키세요.

## ❌ Don't

- 단순 알림에는 Toast를 사용하세요. Modal은 과합니다.
- 모달 위에 또 다른 모달을 열지 마세요. (중첩 모달)
- `closeOnOverlayClick`을 끄면 사용자가 갇힌 느낌을 받습니다. 꼭 필요한 경우에만 사용하세요.
- 모달 안에 너무 많은 콘텐츠를 넣지 마세요. 스크롤이 필요하면 별도 페이지를 고려하세요.
- 페이지 로드 즉시 모달을 열지 마세요.


## Size 선택 기준

| Size | 사용 맥락 |
|------|----------|
| `sm` | 삭제 확인, 간단한 경고 |
| `md` | 기본값, 폼 입력 |
| `lg` | 복잡한 폼, 미리보기 |
| `xl` | 대시보드, 상세 편집 |
| `full` | 모바일 전체 화면, 에디터 |


## 접근성

- `role="dialog"`와 `aria-modal="true"`가 자동 설정됩니다.
- `title`이 있으면 `aria-labelledby`로 연결됩니다.
- 모달 열릴 때 첫 번째 포커스 가능 요소로 포커스 이동이 필요합니다.
- 모달 외부 콘텐츠는 `aria-hidden="true"`로 처리하세요.
- 모달 닫힐 때 트리거 버튼으로 포커스를 복원하세요.
