# Typography

텍스트 콘텐츠를 일관되게 표현하기 위한 컴포넌트입니다.

---

## 컴포넌트 구성

| 컴포넌트 | 역할 |
|---------|------|
| `Heading` | 제목 계층 구조 (h1~h6) |
| `Text` | 본문, 설명, 캡션 등 모든 텍스트 |

---

## Heading 사용 규칙

### ✅ Do

- **페이지 내 h1은 하나만** 사용하세요.
- 제목 계층을 **순서대로** 사용하세요. h1 → h2 → h3 순서를 지키세요.
- 시각적 크기가 아닌 **의미적 계층**에 맞게 태그를 선택하세요. 크기가 작아야 하면 `as` prop으로 조정하세요.

### ❌ Don't

- 시각적 강조를 위해 heading 태그를 건너뛰지 마세요. (h1 → h3)
- 단순 굵게 표시를 위해 Heading을 사용하지 마세요. `Text weight="bold"`를 사용하세요.

---

## Text Variant 선택 기준

| Variant | 사용 상황 |
|---------|---------|
| `body` (16px) | 기본 본문, 주요 설명 |
| `body-sm` (14px) | 보조 설명, 폼 레이블 |
| `caption` (12px) | 날짜, 메타 정보, 이미지 캡션 |
| `overline` (12px, 대문자) | 섹션 카테고리 라벨 |

---

## Color 선택 기준

| Color | 사용 상황 |
|-------|---------|
| `primary` | 기본 텍스트 |
| `secondary` | 보조 설명, 덜 중요한 정보 |
| `tertiary` | 플레이스홀더, 힌트 |
| `inverse` | 어두운 배경 위 |
| `success` | 성공 메시지 |
| `danger` | 에러 메시지 |

---

## ✅ Do

- 한 단락 안에서 color와 weight를 **2가지 이상 혼용**하지 마세요.
- `truncate` prop은 고정 너비 컨테이너 안에서 긴 텍스트를 처리할 때 사용하세요.
- `as` prop으로 시각과 시맨틱을 분리할 수 있습니다.

```tsx
// h2처럼 보이지만 h3 태그로 렌더링
<Heading level="h2" as="h3">섹션 제목</Heading>
```

---

## 접근성

- Heading은 페이지 아웃라인을 구성하므로 계층을 올바르게 유지하세요.
- `color` prop은 시각적 색상만 변경하며, 의미는 전달하지 않습니다. 중요한 상태는 텍스트로도 명시하세요.
