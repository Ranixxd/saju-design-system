# Typography

텍스트의 시각적 위계와 가독성을 일관되게 유지하기 위한 컴포넌트입니다.


## 컴포넌트 구성

| 컴포넌트 | 용도 |
|---------|------|
| `Heading` | 제목 (h1~h6) |
| `Text` | 본문 텍스트 |
| `Label` | 폼 레이블, 캡션 |
| `Caption` | 보조 설명, 날짜, 메타 정보 |


## ✅ Do

- 시각적 스타일(`as`)과 의미론적 태그(`level`)를 분리해서 사용하세요.
  - 예: 시각적으로 h3처럼 보이지만 DOM은 h2여야 할 때 → `<Heading level="h2" as="h3">`
- 페이지당 `h1`은 **1개**만 사용하세요.
- 제목 레벨은 순서대로 사용하세요. (h1 → h2 → h3, 건너뛰기 금지)
- 본문 텍스트는 `Text` 컴포넌트를 사용하세요.
- 폼 레이블에는 `Label`을 사용하고 `htmlFor`로 연결하세요.

## ❌ Don't

- 폰트 크기를 키우려고 상위 Heading 태그를 남용하지 마세요.
- 색상만으로 텍스트의 중요도를 구분하지 마세요.
- 본문 텍스트를 `Heading`으로 스타일링하지 마세요.
- 너무 많은 텍스트 스타일을 한 화면에 혼용하지 마세요.


## 위계 사용 가이드

```
h1 - 페이지 제목 (1개)
  h2 - 주요 섹션
    h3 - 하위 섹션
      h4 - 세부 항목
```


## 접근성

- 제목 레벨(`h1`~`h6`)은 시각적 스타일이 아닌 **문서 구조**를 따라야 합니다.
- 스크린리더는 제목 레벨로 페이지 내 탐색을 합니다.
- `Caption`이나 `Label`은 색상 대비 4.5:1 이상을 유지하세요.
- 이탤릭체, 대문자 텍스트는 스크린리더가 강조로 인식하지 않습니다.
